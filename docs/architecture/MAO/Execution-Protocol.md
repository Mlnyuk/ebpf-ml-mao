# 단계별 실행 프로토콜

이 문서는 Step 단위 구현을 어떤 순서로 진행할지에 대한 표준 프로토콜이다.

## 1. Phase 0: 작업 선언

반드시 먼저 아래를 확정한다.

- 이번 단계 번호
- 목표
- 범위
- 제외 범위
- 산출물
- 검증 방법

## 2. Phase 1: 사전 탐색

해야 할 일:

- 현재 저장소 구조 확인
- 관련 문서 읽기
- 기존 구현 재사용 가능 부분 확인
- 경로 참조가 어디에 있는지 검색

## 3. Phase 2: 외부 에이전트 호출

- Claude Code: 설계 초안
- Gemini CLI: 탐색/리스크 점검

규칙:

- 외부 응답은 참고 자료다.
- 최종 구현 계획은 메인 실행자가 다시 요약해야 한다.
- 응답이 늦으면 로컬 작업을 먼저 진행한다.

## 4. Phase 3: 구현

해야 할 일:

- 디렉터리 생성
- 코드 수정
- 테스트 추가
- 샘플 입력 추가
- 단계 문서 초안 작성

## 5. Phase 4: 검증

최소로 해야 할 것:

- 단위 테스트
- CLI 실행 또는 샘플 실행
- 출력 산출물 확인
- 경로/링크 점검

## 6. Phase 5: 문서화

각 단계는 최소 아래 문서를 남긴다.

- `docs/steps/stepN/README.md`
- 필요 시 `docs/steps/stepN/agent-notes.md`

## 7. Phase 6: 저장소 정리

마지막에 확인할 것:

- 샘플은 `samples/stepN/` 아래에 있는가
- 단계 문서는 `docs/steps/stepN/` 아래에 있는가
- 배포 파일은 `deploy/` 아래에 있는가
- 캐시/아티팩트가 Git에 안 올라가는가

## 8. Phase 7: 반영

반드시 이 순서로 간다.

1. `git status`
2. 필요한 파일만 `git add`
3. 의미 있는 커밋 메시지 작성
4. `git push`

# 2. Agent Role Definition

이 프로젝트의 에이전트는 `문제 해결 실행기`가 아니라 `분석 협업자`여야 한다. 역할은 아래 네 개로 고정하는 것이 MVP와 확장성 사이 균형이 좋다.

## 1. Summarizer Agent

목적:
- 원시 이벤트를 짧고 밀도 높은 사건 요약으로 압축

입력:
- eBPF/Tetragon 이벤트 묶음
- 메트릭 스냅샷
- 최근 시간 윈도우 메타데이터

출력:
- 이벤트 요약
- 핵심 엔터티 목록
- 의심 행위 후보

책임:
- 이벤트 폭주를 줄인다.
- 동일 프로세스, Pod, 노드 기준으로 묶는다.
- 이후 에이전트가 읽을 수 있는 서술형 컨텍스트를 만든다.

잘하는 일:
- 긴 로그 묶음 압축
- 반복 패턴 제거
- 핵심 타임라인 추출

## 2. Analyst Agent

목적:
- ML 추론 결과를 해석하고 이상 징후의 의미를 설명

입력:
- feature vector 또는 집계 결과
- anomaly score
- baseline 대비 편차 정보
- Summarizer 결과

출력:
- 이상 여부 판단 초안
- 설명 가능한 원인 후보
- 점수 근거 설명

책임:
- 단순히 점수가 높다고 말하지 않고 왜 높은지 설명한다.
- false positive 가능성을 적는다.
- 특징량 기준으로 어떤 이상 패턴이 감지됐는지 정리한다.

잘하는 일:
- 수치 해석
- 기준선 대비 변화 설명
- 모델 출력의 의미 부여

## 3. Correlator Agent

목적:
- 이벤트, 메트릭, ML 결과를 교차 검증해서 맥락을 만든다.

입력:
- Summarizer 결과
- Analyst 결과
- 프로세스, 컨테이너, 노드, 네트워크 메타데이터

출력:
- 상관분석 리포트
- 공격/장애 시나리오 후보
- 추가 확인이 필요한 지점

책임:
- 개별 신호를 하나의 사건으로 연결한다.
- 프로세스 실행, 파일 접근, 네트워크 연결, CPU/메모리 변화 사이 관계를 묶는다.
- 단일 이벤트가 아니라 연속 행위 시퀀스를 본다.

잘하는 일:
- cross-signal reasoning
- kill chain 유사 흐름 추론
- 서로 다른 데이터 소스 연결

## 4. Reviewer Agent

목적:
- 앞선 에이전트들의 판단을 검토하고 최종 결과를 정리

입력:
- Summarizer, Analyst, Correlator 결과 전체

출력:
- 최종 anomaly report
- confidence
- 추가 수집 또는 재분석 요청

책임:
- 충돌하는 결론을 정리한다.
- 근거 부족 시 `보류`를 선언한다.
- 최종 결과를 사용자 친화적으로 출력한다.

잘하는 일:
- 합의 형성
- 판단 보수화
- 결과 포맷팅

## 오케스트레이션 규칙

- `Summarizer`는 항상 선행한다.
- `Analyst`와 `Correlator`는 Summarizer 결과를 공유받아 병렬 수행할 수 있다.
- `Reviewer`는 마지막에만 실행한다.
- confidence가 낮으면 pipeline은 추가 수집을 한 번만 재시도한다.

## 권장 모델 배치

- `Summarizer`: 긴 문맥 요약에 강한 모델
- `Analyst`: 수치 해석과 구조적 설명이 안정적인 모델
- `Correlator`: 복수 신호 추론이 강한 모델
- `Reviewer`: 보수적 판단과 일관성 검증이 강한 모델

핵심은 특정 벤더명이 아니라 `역할 고정`이다. 구현체는 바뀌어도 역할 계약은 유지해야 한다.
